## 34:00 到底開發上會發生什麼事？
![developCost](./pic/developPressure.png)
若實踐瀑布式開發，最終會產生隨時間增加，而開發成本到後期劇增的問題 \
但若是邊寫邊測試，邊寫邊交付，則開發成本是可控的成長，不會驟然提升

遽增主因：
程式複雜度提升，導致程式開發難度增加

解決辦法：
撰寫程式/重構/測試，三者一體的循環 \
當程式複雜度提升時，藉由既有的測試程式，安心進行重構 \
重構後的程式複雜度下降，則可恢復繼續開發
